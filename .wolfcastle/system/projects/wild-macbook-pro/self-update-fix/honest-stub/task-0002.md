# Update the update command to surface unavailable status honestly

The update command in cmd/update.go currently handles three states: error, updated, and already-current. With the stub now reporting unavailability, the command needs a fourth path.

Changes needed:
- In human output mode: print a message like 'Self-update is not available yet. Regenerating system/base/ only.' instead of the misleading 'Already running X. No upgrade needed.'
- In JSON output mode: use update_status 'unavailable' instead of 'unchanged' when the updater reports that checking is not implemented.
- The Reinit() call for system/base/ regeneration should still proceed regardless of update availability, since that part works fine.
- Preserve the existing error-handling path for when a real updater is eventually wired in and encounters transient failures.
