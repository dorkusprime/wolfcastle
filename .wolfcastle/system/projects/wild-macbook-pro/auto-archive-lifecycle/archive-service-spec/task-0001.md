# Write archive service specification

Create a spec document via wolfcastle spec create that defines the complete auto-archive service contract. The spec must cover:

1. **Archive state model**: How completion timestamps are tracked on nodes (field on NodeState and/or RootIndex entries), what 'archived' means in the state machine (a new status? a flag? removal from active index with metadata preserved?).

2. **File layout**: Where archived node state lives. The scope says 'system/projects/ to archive/' but clarify whether this means the .wolfcastle/archive/ directory (which currently holds markdown rollup entries) or a new .wolfcastle/system/archive/ directory. Define the directory structure for archived state (preserving the node hierarchy or flattening).

3. **Operations**: move (archive a completed node), restore (un-archive back to active), delete (permanently remove archived state). Define preconditions (e.g., move requires complete status, restore places node back in not_started or complete state), side effects on RootIndex, and error cases.

4. **Config schema**: An ArchiveConfig struct with at minimum auto_archive_delay_hours (int, default 24), auto_archive_enabled (bool, default true). Where this lives in the Config tree.

5. **Daemon timer interface**: How the daemon discovers eligible nodes (completed top-level orchestrators with last activity older than threshold), the polling interval for archive checks, and interaction with the existing daemon loop (new goroutine? checked in RunOnce?).

6. **Status integration contract**: What the status command needs from the archive system to render counts and filtered views. Define the data structures or query interface.

Reference existing specs: 2026-03-12T00-05Z-archive-format.md (current archive entry format), 2026-03-12T00-00Z-state-machine.md (node lifecycle). Reference ADR-016 (archive format with deterministic rollup).
