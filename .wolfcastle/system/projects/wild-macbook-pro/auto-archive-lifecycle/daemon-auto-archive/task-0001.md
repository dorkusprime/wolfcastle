# Implement auto-archive scan in daemon

Add an auto-archive check to the daemon loop. The check should:

1. Read the ArchiveConfig from the daemon's config to determine if auto-archive is enabled and what the delay threshold is.
2. Scan the RootIndex for top-level orchestrator nodes in 'complete' state.
3. For each complete node, compare the completion timestamp (or last activity time from breadcrumbs/audit) against the current time minus the delay threshold.
4. For eligible nodes, call the archive service's Move operation.
5. Log each auto-archive action.

Decide whether this runs inline in RunOnce() (checked each iteration), on a separate timer (like inbox polling), or as a periodic check at a configurable interval. Document the decision in an ADR if it introduces a new goroutine.

Reference the inbox goroutine pattern in internal/daemon/stages.go for the parallel goroutine approach. Reference the daemon's existing timer infrastructure (poll intervals in DaemonConfig).
