# Implement config validate command

Create cmd/config/validate.go with a newValidateCmd(app) function following the existing command pattern (see show.go, set.go for reference). The command should:

1. Load the three-tier config via app's ConfigRepository.Load()
2. Run config.ValidateStructure(cfg) to check structural constraints
3. Optionally run config.Validate(cfg) for full validation (with a --full flag)
4. Collect errors and warnings into a structured report
5. Human output: print each issue with severity, category, and description. Print a summary line (e.g., '2 errors, 1 warning'). Use output.PrintHuman for normal output, output.PrintError for errors
6. JSON output: emit a structured envelope via output.Print(output.Ok(...)) containing the issues list, error count, and warning count
7. Exit code: return nil (exit 0) when no errors found, return an error (non-zero) when errors exist. Warnings alone should not cause non-zero exit
8. Register the command in cmd/config/register.go's Register() function

Reference the existing command pattern in cmd/config/show.go and the validation functions in internal/config/validate.go.
