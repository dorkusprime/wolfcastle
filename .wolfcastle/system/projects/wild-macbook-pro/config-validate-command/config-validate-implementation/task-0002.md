# Write tests for config validate command

Create cmd/config/validate_test.go with comprehensive tests following the test pattern in cmd/config/show_test.go. Use testutil.NewEnvironment to set up a temporary .wolfcastle directory. Tests should cover:

1. Valid config: command exits 0, human output shows 'no issues' or similar clean message, JSON output has empty issues array and zero counts
2. Config with structural errors (e.g., empty pipeline, invalid stage references): command exits non-zero, human output lists each error, JSON output contains the errors
3. Config with warnings but no errors: command exits 0 (warnings are non-fatal)
4. The --full flag runs full validation including identity checks
5. Missing or unreadable config: command returns an appropriate error
6. Both human and JSON output modes for each scenario

Maintain >90% coverage on the new validate.go file.
