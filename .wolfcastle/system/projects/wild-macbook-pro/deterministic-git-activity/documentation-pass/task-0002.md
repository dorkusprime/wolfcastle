# Document git config section in configuration.md

Update docs/humans/configuration.md (and any new config pages created by the config-docs-overhaul project) to document the full git config section:

- git.auto_commit (bool, default: true): master switch for all daemon git operations
- git.commit_on_success (bool, default: true): commit after successful task completion
- git.commit_on_failure (bool, default: true): commit partial work after task failure
- git.commit_state (bool, default: true): include .wolfcastle/ state in commits
- git.skip_hooks (bool, default: false): pass --no-verify to git commit

Explain the hierarchy: auto_commit is the master switch, the others are subordinate. When auto_commit is false, no commits happen regardless of other settings.

Include examples of common configurations:
- Disable all auto-commits: git.auto_commit: false
- Commit on failure only (for manual commit workflows): commit_on_success: false
- Exclude state from commits: commit_state: false
