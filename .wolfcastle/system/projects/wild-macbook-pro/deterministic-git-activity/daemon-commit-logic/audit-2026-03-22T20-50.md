## Summary

Refactored the daemon's post-iteration commit logic from a single autoCommitPartialWork function into a config-driven commitAfterIteration that respects auto_commit, commit_on_success, commit_on_failure, and commit_state flags, with spec-compliant commit messages. Implemented staging area preservation using a separate GIT_INDEX_FILE so daemon commits never touch the user's .git/index. Added commitDirect as a fallback when temp index creation fails. The implementation is backed by 14 new unit and integration tests covering all flag combinations, plus 4 tests verifying byte-level index preservation, working tree safety, temp file cleanup, and clean-tree skip behavior. An ADR documents the separate-index decision over the stash-based alternative. Audit found no correctness, security, or concurrency issues; all findings were cosmetic.
