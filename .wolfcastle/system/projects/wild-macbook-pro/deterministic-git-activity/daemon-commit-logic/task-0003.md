# Implement staging area preservation

The current autoCommitPartialWork runs 'git add -u' and 'git add .wolfcastle/' which can clobber the user's manually staged changes. Implement staging area preservation per the spec. The preferred approach: use a temporary GIT_INDEX_FILE for daemon commits, leaving the user's index untouched. The alternative (stash-based) approach is: (1) git stash --keep-index --quiet, (2) git add ..., (3) git commit, (4) git stash pop --quiet. The spec notes the stash approach may be fragile; prefer the separate index approach if feasible. Whichever approach is chosen, document the decision in an ADR via 'wolfcastle adr create'.
