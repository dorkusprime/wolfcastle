# Update config-reference.md with new git fields

Update docs/humans/config-reference.md to document the three new git config fields in the git section (currently lines 388-428):

- git.commit_on_success: boolean, default true. Commit after successful task completion. Only applies when auto_commit is true.
- git.commit_on_failure: boolean, default true. Commit partial work after task failure. Only applies when auto_commit is true.
- git.commit_state: boolean, default true. Include .wolfcastle/ state directory in daemon commits. When false, only code changes are committed. Only applies when auto_commit is true.

Follow the existing format in the config reference for consistency.
