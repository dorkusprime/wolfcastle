# Add new GitConfig fields to types.go

Add three new fields to the GitConfig struct in internal/config/types.go: CommitOnSuccess (bool, json:commit_on_success), CommitOnFailure (bool, json:commit_on_failure), and CommitState (bool, json:commit_state). CommitOnSuccess and CommitOnFailure control whether the daemon commits after successful or failed task iterations respectively, and only apply when AutoCommit is true. CommitState controls whether .wolfcastle/ state is included in commits. All default to true. See spec section 'Config' for the full schema. Consider whether SkipHooksOnAutoCommit should be renamed to match the spec's skip_hooks naming, but if renaming would break existing configs, keep both and handle via migration.
