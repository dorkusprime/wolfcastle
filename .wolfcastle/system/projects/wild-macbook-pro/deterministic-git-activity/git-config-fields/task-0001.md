# Add new git config fields to GitConfig struct

Add three new boolean fields to the GitConfig struct in internal/config/types.go:

- CommitOnSuccess (json: commit_on_success) - controls whether daemon commits after successful task completion
- CommitOnFailure (json: commit_on_failure) - controls whether daemon commits partial work after task failure
- CommitState (json: commit_state) - controls whether .wolfcastle/ state is included in daemon commits

These join the existing AutoCommit (master switch) and SkipHooksOnAutoCommit fields. When AutoCommit is false, the fine-grained controls are ignored.

Also update Defaults() in internal/config/config.go to set all three new fields to true by default.

Note: the spec calls for a skip_hooks field, but SkipHooksOnAutoCommit already serves this purpose. Do not rename or duplicate it.
