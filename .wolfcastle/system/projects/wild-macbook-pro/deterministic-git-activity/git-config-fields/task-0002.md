# Wire defaults and validation for new git config fields

In internal/config/config.go, update the Defaults() function to set CommitOnSuccess: true, CommitOnFailure: true, CommitState: true. In Validate() (or wherever config validation lives), ensure the fine-grained fields are ignored when AutoCommit is false. Add a migration in the migration service so that existing config files missing the new fields get sensible defaults applied (all true) rather than Go zero values (all false, which would silently disable commits).
