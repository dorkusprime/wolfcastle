# Add daemon commit tests

Write tests for the new daemon commit behavior:

- Test commit-on-success: after WOLFCASTLE_COMPLETE, verify a commit is created with the expected message format.
- Test commit-on-failure: after task failure, verify a commit is created with the partial work message.
- Test auto_commit: false disables all commits (neither success nor failure path commits).
- Test commit_on_success: false skips the success commit but still commits on failure.
- Test commit_on_failure: false skips the failure commit but still commits on success.
- Test commit_state: false excludes .wolfcastle/ from the commit.
- Test staging area preservation: manually stage a file, run a daemon commit, verify the user's staged file is still staged (not committed by the daemon).
- Test clean working tree: no commit is created when there are no changes.
- Test skip_hooks: verify --no-verify is passed when skip_hooks is true.

Follow existing test patterns in the daemon package. These will likely need a real git repo (testutil fixtures) rather than mocks, per the project's testing philosophy.
