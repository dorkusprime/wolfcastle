# Add new config fields to GitConfig struct

Add three new fields to the GitConfig struct in internal/config/types.go:
- CommitOnSuccess (bool, json: commit_on_success): controls whether the daemon commits after successful task completion. Default: true.
- CommitOnFailure (bool, json: commit_on_failure): controls whether the daemon commits partial work after task failure. Default: true.
- CommitState (bool, json: commit_state): controls whether .wolfcastle/ state is included in daemon commits. Default: true.

All three are subordinate to the existing AutoCommit master switch (ignored when auto_commit is false).

Also rename SkipHooksOnAutoCommit to SkipHooks (json: skip_hooks) per the spec, or confirm whether the existing field name should remain. Check whether a rename would break existing user configs and decide accordingly.

Update Defaults() in internal/config/config.go to set the new defaults.
Update Validate() in internal/config/validate.go if any validation is needed (the bool fields likely need no validation, but confirm CommitOnSuccess/CommitOnFailure aren't both false when auto_commit is true, which would be a no-op config worth warning about).

Reference the spec at .wolfcastle/docs/specs/2026-03-22T07-00Z-deterministic-git.md for the full config schema.
