# Author writing.md and writing/voice.md

Create two files:

1. internal/project/templates/prompts/classes/writing.md — The writing discipline class prompt. 30-60 lines. Covers: reader-first structure, concrete examples over abstract description, scannable formatting, progressive disclosure (lead with the answer, details follow), tone calibration. Must include a reference to writing/voice.md for voice-specific guidance (e.g., 'For voice and tone guidance, see the voice prompt loaded alongside this one' or similar). Research current technical writing best practices (Google developer docs style guide, Microsoft writing style guide, etc.).

2. internal/project/templates/prompts/classes/writing/voice.md — Voice guidance separated so users can override it independently. 20-40 lines. Covers: default voice characteristics (clear, direct, human), when to adjust formality, how to handle jargon, audience awareness. This file exists so organizations can drop in their own voice guide without touching the writing discipline prompt.

ZERO wolfcastle-specific content in either file.
