# Wire universal.md injection into ContextBuilder.Build

Update ContextBuilder.Build in internal/pipeline/context_builder.go to always inject universal.md content under a '## Universal Guidance' heading, regardless of whether the task has a Class field set.

Current behavior (lines 111-121): class guidance is only injected when task.Class != "". universal.md is never loaded.

Required behavior per the task-classes spec (Prompt Assembly section):
1. Always resolve and inject prompts/classes/universal.md under '## Universal Guidance'.
2. When task.Class is empty: also inject prompts/classes/coding/default.md under '## Class Guidance'.
3. When task.Class is set and resolves: inject the resolved class prompt under '## Class Guidance'.
4. When task.Class is set but fails to resolve: fall back to coding/default.md under '## Class Guidance'.

The resolution should use ClassRepository for classified tasks and PromptRepository.ResolveRaw for universal.md and coding/default.md (since those aren't keyed by class config).

Reference the ContextBuilder contract spec and the task-classes spec for the exact assembly order.
