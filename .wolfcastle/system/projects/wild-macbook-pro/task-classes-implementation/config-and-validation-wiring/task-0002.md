# Wire ClassRepository.Validate() into daemon startup

After the daemon creates the ClassRepository and calls Reload(cfg.TaskClasses) in internal/daemon/daemon.go (currently lines 115-116), call classes.Validate() and log warnings for any class keys whose prompt files are missing.

The validation should:
1. Call classes.Validate() which returns []string of missing class keys
2. If the slice is non-empty, log a warning (not an error) listing the missing keys
3. Do NOT block daemon startup on missing prompts (graceful degradation)

Use the daemon's existing logger. The message should be clear enough that a user can understand which class prompts they need to create (e.g., "task classes with missing prompt files: [go, python]").

Also update daemon_test.go to verify that startup succeeds both with and without missing class prompts, and that the warning is logged when prompts are missing.
