# Add --class validation to task add CLI

Update the task add command in cmd/task/add.go to validate the --class flag value against the config's task_classes map.

Currently (line ~60), the --class value is accepted as any string and stored directly on the task. Change this to:
1. Load the config (the command already has access to the app context which includes config)
2. If --class is provided and non-empty, check that the value exists as a key in cfg.TaskClasses
3. If the key is not found, return an error listing the valid class values (from ClassRepository.List() or sorted keys of cfg.TaskClasses)
4. The error message should be helpful: "unknown task class 'foo'; valid classes: [audit, coding/go, coding/python, ...]"

Also add tests in cmd/task/add_test.go (or the appropriate test file) that verify:
- Valid class values are accepted
- Invalid class values are rejected with a helpful error
- Empty/omitted --class continues to work (no validation when not provided)
