# Remove stageLabels identity map

In internal/logrender/record.go, the stageLabels map (lines ~31-39) maps every key to itself. The StageLabel() method (lines ~41-47) does a lookup and falls back to r.Stage on miss, which means the map is a no-op: it returns the same value for known stages as the fallback does for unknown stages. Delete the stageLabels map variable and simplify StageLabel() to just 'return r.Stage'. Update record_test.go TestStageLabel accordingly: the test table can shrink since there is no longer a distinction between known/unknown stages, but keep at least one case verifying that StageLabel returns the raw Stage value.
