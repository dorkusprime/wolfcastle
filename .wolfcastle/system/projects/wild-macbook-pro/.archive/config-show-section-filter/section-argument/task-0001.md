# Implement section filtering in config show

Modify cmd/config/show.go to accept an optional positional argument (the section name). After resolving the config through any of the three existing code paths (default merged, --tier, --raw), convert the result to map[string]any if needed, then extract the named key. If the key exists, output only its value. If the key does not exist, return an error listing the valid section names. The argument should work with --json output as well, wrapping the filtered value in the standard envelope. Update the command's Use string to 'show [section]' and add the section argument to the Long description. Use cobra.MaximumNArgs(1) to allow zero or one positional argument.
