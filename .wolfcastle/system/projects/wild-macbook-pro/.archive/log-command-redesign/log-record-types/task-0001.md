# Add iteration_start record to daemon startup

The daemon needs to emit an iteration_start record at the beginning of each iteration. This record is the primary mechanism for session boundary detection in the log command. The record should contain: type='iteration_start', the node address being worked on, the stage type (execute, intake, plan, audit, etc.), and the iteration number. The logging package's StartIteration/StartIterationWithPrefix methods are the natural place to emit this record. Check whether the daemon already logs sufficient data for the log command to reconstruct stage_start node addresses and stage types. If stage_start records are missing the node address field, add it.
