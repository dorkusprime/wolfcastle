# Implement summary renderer for replay mode

Create internal/logrender/summary.go. Implement a SummaryRenderer struct that accepts an io.Writer for output. The Replay(records <-chan Record) method consumes records and produces the summary format: one line per completed stage. The renderer must track stage_start records to pair them with stage_complete records for duration calculation. Output format per the spec: '[stage-label] node-address (duration)' with column alignment. The stage label comes from StageLabel(). Node address includes the task path when present (node/task-id). Use ✓ for exit_code 0 and ✗ for non-zero. For audit_report_written records, print the report path indented on the next line: '  report: <path>'. planning_start/planning_complete should be rendered as [plan] stages. Skip terminal_marker, assistant, and other record types. Filter out daemon_start/daemon_stop records (they are not stages).
