# Define the Record struct and NDJSON line parser

Create internal/logrender/record.go. Define a Record struct with typed fields for every NDJSON record the renderers consume: Type, Timestamp (time.Time parsed from RFC3339), Level, Trace, Stage, Node, Task, ExitCode, Text, Path, Marker, Error, and a raw map[string]any fallback for forward compatibility. Write a ParseRecord(line string) (Record, error) function that unmarshals a JSON line into the struct. The parser must never panic on malformed input; return an error instead. Include a StageLabel() method that maps stage field values to display labels per the spec: intake->intake, plan->plan, execute->execute, audit->audit, remediate->remediate, spec-review->spec-review. Records with unrecognized type fields should parse without error (the renderers will skip them).
