# Implement record reader for replay and follow modes

Create internal/logrender/reader.go. This provides the I/O abstraction that renderers use to consume records. Implement two modes: (1) ReplayReader: reads all NDJSON lines from a slice of file paths (a session's files in order), parses each into a Record, and delivers them via a channel or iterator. Handles both .jsonl and .jsonl.gz files. (2) FollowReader: tails the active log directory, detecting new files as iterations advance, parsing new lines as they're appended, and delivering Records via a channel. Accepts a context.Context for cancellation. Uses polling (not fsnotify) with a configurable interval (default 200ms) to check for new content and new files. Both readers skip malformed lines silently (log a debug record internally if needed, but never surface parse errors to the renderer). The reader interface should be simple enough that renderers don't need to know whether they're in replay or follow mode: they just range over records.
