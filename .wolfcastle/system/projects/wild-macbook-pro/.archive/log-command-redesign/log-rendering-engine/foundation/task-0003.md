# Implement session detection and log file enumeration

Create internal/logrender/session.go. A session is one daemon run: it starts with an iteration-1 log file and includes all consecutive iteration files until the next iteration-1 file or end of directory. Log files follow the naming convention NNNN-YYYYMMDDTHH-MMZZ.jsonl (the iteration number is the leading 4-digit segment). Implement: (1) ListSessions(logDir string) ([]Session, error) that scans the log directory, groups files into sessions by finding iteration-1 boundaries, and returns sessions ordered most-recent-first. A Session struct holds the list of file paths in iteration order. (2) ResolveSession(logDir string, index int) (Session, error) where index 0 is the latest session, 1 is the previous, etc. Returns an error if the index is out of range. Handle .jsonl.gz compressed files transparently (the logging package compresses old iterations). Use the existing logging.LatestLogFile as reference for file naming conventions but do not depend on it at runtime.
