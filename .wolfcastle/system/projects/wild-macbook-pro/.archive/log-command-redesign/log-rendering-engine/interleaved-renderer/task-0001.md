# Implement interleaved renderer

Create internal/logrender/interleaved.go. Implement an InterleavedRenderer struct that accepts an io.Writer. The Render(ctx context.Context, records <-chan Record) method processes records chronologically and produces interleaved output. Stage events (stage_start, stage_complete, planning_start, planning_complete) print as timestamped header lines: 'HH:MM:SS ▶ [stage-label] node-address' for starts, 'HH:MM:SS ✓ [stage-label] node-address (duration)' for completions. The timestamp is the record's wall-clock time formatted in local timezone. Assistant records print indented with a timestamp: 'HH:MM:SS     text'. The 4-space indent visually separates thoughts from stage headers per the spec. audit_report_written records print indented below the audit completion. Duration calculation requires tracking stage_start timestamps the same way the summary renderer does. The renderer must buffer start times keyed by (stage, node, task). Skip terminal_marker, daemon_start, daemon_stop, and other non-display record types. Works for both replay and follow since it processes records as they arrive.
