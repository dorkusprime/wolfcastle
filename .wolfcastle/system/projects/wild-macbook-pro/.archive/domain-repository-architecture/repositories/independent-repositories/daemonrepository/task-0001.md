# Implement DaemonRepository struct and constructor

Create internal/daemon/repository.go with:

- DaemonRepository struct holding a single systemDir field (derived as wolfcastleRoot + "/system" in the constructor).
- NewDaemonRepository(wolfcastleRoot string) *DaemonRepository constructor.
- ReadPID() (int, error) — reads PID from systemDir/wolfcastle.pid, trims whitespace, converts to int.
- WritePID(pid int) error — writes PID as decimal with trailing newline.
- RemovePID() error — removes PID file; returns nil on os.ErrNotExist.
- HasStopFile() bool — checks existence of systemDir/stop.
- WriteStopFile() error — creates the stop file (empty, 0644).
- RemoveStopFile() error — removes stop file; returns nil on os.ErrNotExist.
- LogDir() string — returns systemDir/logs path (escape hatch for Logger).

Reference the existing pid.go for PID logic and daemon.go for stop file logic. The new type consolidates these scattered path constructions into one struct.

Do NOT remove the existing free functions (pid.go) yet — that happens in a later migration step when callers are updated.
