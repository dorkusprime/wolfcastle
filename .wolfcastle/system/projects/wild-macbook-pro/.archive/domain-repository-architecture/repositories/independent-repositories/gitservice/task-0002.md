# Test git.Service with real git repos

Create internal/git/git_test.go with integration tests that create real temporary git repos:

1. TestService_IsRepo: Init a temp git repo, verify IsRepo returns true. Also verify IsRepo returns false for a non-repo directory.
2. TestService_CurrentBranch: Init repo, verify returns "main" (or whatever default). Create and checkout a branch, verify it returns the new branch name.
3. TestService_CurrentBranch_EmptyRepo: Init a repo with no commits, verify CurrentBranch still returns a branch name (the fallback path).
4. TestService_HEAD: Init repo, make a commit, verify HEAD returns a 40-char hex string. Verify HEAD returns empty string for a non-repo.
5. TestService_IsDirty: Start clean, verify false. Create an untracked file, verify true. Test the excludePaths parameter by creating a file under an excluded path.
6. TestService_HasProgress: Record HEAD after initial commit, verify HasProgress is false. Make a new commit, verify HasProgress is true against the old HEAD. Also test dirty-but-no-new-commits case (should return true).
7. TestService_Worktree: Create a worktree, verify the directory exists and is a git checkout. Remove it, verify cleanup.

Use t.TempDir() for repo directories. Use git init, git add, git commit via exec.Command to set up test state.
