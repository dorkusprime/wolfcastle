# Implement NodeState.RenderContext method

Add a RenderContext(taskID string) string method to NodeState in internal/state/render.go. This method extracts the node-level rendering from pipeline/context.go lines 55-57 and 182-187 into the domain type.

The method should render:
- Node metadata: '**Node:** <address>' requires the node to know its address. Since NodeState has an ID field but not a full address, and the spec signature is RenderContext(taskID string), the node address must come from elsewhere. For now, render the node's ID, Name, Type, and State. The full address will be provided by ContextBuilder when it composes the final context.
- Linked specs under a '## Linked Specs' header when specs are non-nil/non-empty
- Node type and state information

The method renders node-level context that surrounds the task-specific rendering. The ContextBuilder will call ns.RenderContext(taskID) to get the node framing, then task.RenderContext() for task detail, then ns.Audit.RenderContext() for audit info.

Do not modify callers or remove the existing buildIterationContext function. This task only adds the new method and its tests.
