# Refactor Task.RenderContext signature

Refactor the existing Task.RenderContext method in internal/state/render.go to match the spec signature: RenderContext() string (no parameters).

Current signature: RenderContext(nodeAddr string, nodeDir string) string
Target signature: RenderContext() string

Changes required:
1. Remove nodeAddr parameter. The task address line ('**Task:** project/auth/task-0001') currently uses nodeAddr to build the full path. Change this to just use the task's own ID ('**Task:** task-0001'). The full address will be provided by NodeState.RenderContext or ContextBuilder.

2. Remove nodeDir parameter and the .md file reading block (lines 27-35 of current render.go). Reading per-task .md files from the filesystem is I/O that belongs in the ContextBuilder, not on the domain type's render method. The ContextBuilder has access to file paths and can prepend .md content before calling task.RenderContext().

3. Update all callers. The existing render_test.go tests call task.RenderContext('project/auth', '') — these must be updated to call task.RenderContext(). Search for all callers with: grep -r 'RenderContext(' internal/ to find them.

4. Update the existing tests in render_test.go to reflect the new signature and output format.
