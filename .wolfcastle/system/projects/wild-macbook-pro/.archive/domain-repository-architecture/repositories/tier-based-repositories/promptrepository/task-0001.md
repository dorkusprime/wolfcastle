# Write PromptRepository contract spec

Write a contract spec for PromptRepository following the established contract spec pattern. The spec must define:

- The PromptRepository struct with a tiers field (tierfs.Resolver)
- Dual constructors: NewPromptRepository(wolfcastleRoot string) constructs tierfs.FS internally; NewPromptRepositoryWithTiers(tiers tierfs.Resolver) accepts an injected resolver for testing
- Resolve(name string, ctx any) (string, error): resolves a prompt template by short name (e.g. "execute" maps to prompts/execute.md), reads content via tiers.Resolve("prompts/"+name+".md"), then executes as Go text/template with ctx. If ctx is nil, returns raw content without template execution. Wraps template parse/execute errors
- ResolveRaw(category string, name string) (string, error): resolves raw content by category and name (e.g. category="prompts", name="execute.md") via tiers.Resolve(category+"/"+name). Returns string content, no template processing
- ListFragments(category string, include []string, exclude []string) ([]string, error): collects all .md fragments in a category (e.g. "rules") across tiers with override semantics using tiers.ResolveAll(category). Applies include/exclude filtering. Returns file contents as string slice, sorted by filename
- WriteBase(relPath string, data []byte) error: delegates directly to tiers.WriteBase(relPath, data)
- WriteAllBase(templates fs.FS) error: walks an fs.FS and writes each file to the base tier via tiers.WriteBase. Used during scaffold to seed default prompts from embedded templates
- Error wrapping convention: all errors prefixed with "prompts:"

Study the existing ResolveFragment, ResolveAllFragments, and ResolvePromptTemplate functions in internal/pipeline/fragments.go. These implement the same logic using raw filepath operations and the hardcoded Tiers variable. The repository version delegates to tierfs.Resolver instead. The existing pipeline.Tiers variable and standalone functions should NOT be modified — they'll be cleaned up in a later migration step.
