# Write ConfigRepository contract spec

Write a contract spec for ConfigRepository following the pattern established by existing contract specs (daemonrepository-contract, git-provider-contract, identity-domain-type-contract). The spec must define:

- The ConfigRepository struct with a tiers field (tierfs.Resolver) and root field (string)
- Dual constructors: NewConfigRepository(wolfcastleRoot string) constructs tierfs.FS internally; NewConfigRepositoryWithTiers(tiers tierfs.Resolver, root string) accepts an injected resolver for testing
- Load() (*Config, error): starts with Defaults() as map, overlays each tier's config.json via DeepMerge, marshals back to *Config, validates with ValidateStructure. Uses tiers.TierDirs() to locate config files rather than hardcoded paths. Missing tier files are not errors. Permission errors propagate
- WriteBase(cfg *Config) error: marshals full *Config to JSON and writes via tiers.WriteBase("config.json", data)
- WriteCustom(data map[string]any) error: marshals partial overlay to JSON, writes to custom tier's config.json using tiers.TierDirs()[1]
- WriteLocal(data map[string]any) error: marshals partial overlay to JSON, writes to local tier's config.json using tiers.TierDirs()[2]
- Error wrapping convention: all errors prefixed with "config:"

Study the existing config.Load() function in internal/config/config.go and the configTiers variable to understand the current implementation that will be refactored into the repository struct. The key change: Load currently takes wolfcastleDir and constructs paths manually; the repository version uses tierfs.Resolver to locate files.
