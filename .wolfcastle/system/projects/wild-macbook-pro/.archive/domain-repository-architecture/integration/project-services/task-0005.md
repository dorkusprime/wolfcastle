# Write MigrationService and ScaffoldService tests

Write tests for both services using testutil.Environment.

MigrationService tests (internal/project/migration_service_test.go):
- MigrateDirectoryLayout moves flat directories into system/
- MigrateDirectoryLayout is idempotent (no-op when system/ exists)
- MigrateDirectoryLayout handles fresh installation (nothing to move)
- MigrateOldConfig moves root config.json to custom tier
- MigrateOldConfig skips move when custom config already exists
- MigrateOldConfig merges config.local.json into local tier
- MigrateOldConfig handles missing source files gracefully

ScaffoldService tests (internal/project/scaffold_service_test.go):
- Init creates complete directory structure
- Init writes correct config files to each tier
- Init creates namespace project directory
- Init extracts embedded prompts
- Reinit regenerates base tier
- Reinit preserves custom and local config
- Reinit calls migrations before regenerating

Use testutil.Environment for repository access. For migration tests, manually create old-layout directories/files before calling migrate methods.
