# Write ContextBuilder tests

Write tests for ContextBuilder in internal/pipeline/context_builder_test.go using testutil.Environment.

Test cases:
- Build includes node context (type, state) from NodeState.RenderContext
- Build includes task context (metadata, deliverables, criteria) from Task.RenderContext
- Build includes class guidance when task has a class set
- Build omits class section when task has no class
- Build includes audit context (breadcrumbs, scope) from AuditState.RenderContext
- Build includes summary requirement when task is the last incomplete task
- Build omits summary requirement when other tasks remain
- Build includes failure header when task has failures
- Build includes decomposition guidance when failure count exceeds threshold
- Build produces empty-ish output for minimal node state (no task found)

Use env.WithProject() to create node state, env.WithPrompt() to seed prompt templates (summary-required.md, context-headers.md, decomposition.md), and env.WithClasses() if available (or manually create ClassRepository).
