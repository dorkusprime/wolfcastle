# Write ContextBuilder contract spec

Create a contract spec for ContextBuilder at .wolfcastle/docs/specs/. The contract should define:

ContextBuilder struct with fields:
- prompts *PromptRepository
- classes *ClassRepository

Constructor: NewContextBuilder(prompts *PromptRepository, classes *ClassRepository)

Method: Build(nodeAddr string, ns *state.NodeState, taskID string, cfg *config.Config) string

Build assembles the full iteration context in this order:
1. Node context via ns.RenderContext(taskID) — type, state, linked specs
2. Task context via task.RenderContext() — metadata, deliverables, criteria, constraints, references
3. Class guidance via classes.Resolve(task.Class) if task has a class set
4. Audit context via ns.Audit.RenderContext() — breadcrumbs, scope
5. Summary requirement prompt (loaded via PromptRepository) if this is the last incomplete task
6. Failure context (loaded via PromptRepository) if task has failures, with decomposition guidance based on failure count and config

Internal helper methods:
- shouldIncludeSummary(ns, taskID, cfg) bool
- renderSummaryRequired() string — loads summary-required.md template
- renderFailureContext(task, cfg) string — loads context-headers.md and optionally decomposition.md

ContextBuilder replaces the monolithic buildIterationContext() function in pipeline/context.go.
