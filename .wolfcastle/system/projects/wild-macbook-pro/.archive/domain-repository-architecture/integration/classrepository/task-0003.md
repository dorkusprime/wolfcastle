# Write ClassRepository tests

Write tests for ClassRepository in internal/pipeline/class_repository_test.go using testutil.Environment.

Test cases:
- Resolve returns correct prompt content for a simple class key
- Resolve applies hierarchical fallback: 'lang-go' falls back to 'lang' when 'lang-go.md' is missing
- Resolve returns error for unknown class key (not in map)
- Resolve returns error when class key exists in map but prompt file missing from all tiers
- Reload replaces class map and subsequent Resolve reflects new classes
- List returns sorted keys
- Validate identifies classes with missing prompt files
- Validate returns empty slice when all classes have prompts

Use env.WithPrompt() to seed prompt files in the base tier. Create a ClassDef map to pass to Reload(). The testutil.Environment already has PromptRepository wired up.
