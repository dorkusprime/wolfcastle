# Write tests for Environment and With* helpers

Create internal/testutil/environment_test.go with tests covering:

NewEnvironment:
- Creates valid directory structure (all three tier dirs exist, projects dir exists, docs/artifacts/archive exist)
- Default config is loadable via config.Load on the created directory
- Identity resolves to test/dev namespace
- State store is functional (can read the empty root index)

NodeSpec builders:
- Leaf() creates correct NodeSpec with leaf type and task IDs
- Orchestrator() creates correct NodeSpec with orchestrator type and children

WithConfig:
- Overrides are merged into custom tier (verify by loading config and checking overridden values)
- Original base tier defaults are preserved for non-overridden keys
- Multiple WithConfig calls accumulate (not replace)

WithProject:
- Creates a simple leaf project with tasks
- Creates a nested orchestrator project with child leaves
- Root index is updated with all nodes
- Node state files are readable and contain correct task entries
- Each node has an audit task

WithPrompt:
- Prompt file is written to correct location
- Content is resolvable via ResolveFragment (or tierfs.Resolve)

WithRule:
- Rule file is written to correct location
- Content is resolvable across tiers

Chaining:
- Verify that multiple With* calls can be chained fluently: NewEnvironment(t).WithConfig(...).WithProject(...).WithPrompt(...)
