# Implement NodeSpec builders and Environment struct

Create internal/testutil/environment.go with the Environment type and NodeSpec builders from the spec.

NodeSpec and builders:
- NodeSpec struct: Name string, Type state.NodeType, Tasks []string, Children []NodeSpec
- Leaf(name string, taskIDs ...string) NodeSpec — creates a leaf NodeSpec
- Orchestrator(name string, children ...NodeSpec) NodeSpec — creates an orchestrator NodeSpec

Environment struct fields (from spec):
- Root string — path to the .wolfcastle/ directory within the temp dir
- Config *config.ConfigRepository (initially nil — will be populated when ConfigRepository exists in a later step)
- Prompts *pipeline.PromptRepository (initially nil — same)
- Classes *pipeline.ClassRepository (initially nil — same)
- Daemon *daemon.DaemonRepository (initially nil — same)
- State *state.StateStore
- Git git.Provider (initially nil — same)
- App *cmdutil.App (initially nil — same)

IMPORTANT: The spec says Environment "evolves across the migration: it starts with just tierfs and gains repository fields as each repository is built." For this initial version:
- The struct should declare all fields so the type signature is stable from the start
- But only populate the fields that have existing implementations: Root, State (via existing StateStore), and a tierfs.FS for internal use
- Repository fields that don't exist yet (Config, Prompts, Classes, Daemon, Git, App) should be left nil with a comment noting they'll be populated when their packages are built
- This means some fields will reference packages that don't exist yet. If this causes import cycle issues, use interface types or leave them as commented-out fields with a TODO. Prefer the approach that compiles cleanly now.

NewEnvironment(t *testing.T) *Environment:
- Creates a temp dir via t.TempDir()
- Constructs a tierfs.FS rooted at <tempdir>/.wolfcastle/system
- Creates the three-tier directory structure (system/base, system/custom, system/local with prompts, rules, audits subdirs)
- Writes default config to base tier, empty {} to custom, test identity to local (same as current SetupWolfcastle but using tierfs.WriteBase where possible)
- Creates projects/<namespace>/ directory and empty root index
- Creates docs/, artifacts/, archive/ directories
- Constructs a StateStore from the projects directory
- Registers t.Cleanup if needed

Reference: the existing SetupWolfcastle in internal/testutil/helpers.go does the same directory/file setup manually. Environment replaces it with a structured approach.
