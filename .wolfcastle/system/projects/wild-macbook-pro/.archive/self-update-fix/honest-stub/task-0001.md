# Update stubUpdater to report unavailability instead of false currency

The stubUpdater in internal/selfupdate/updater.go currently sets AlreadyCurrent: true when Check() or Apply() is called, even though no release channel was queried. This misleads the caller into believing an actual check occurred.

Change the stub behavior:
- Check() should return a Result where AlreadyCurrent is false and Updated is false, plus a new signal that indicates checking is not available. Consider adding a field like 'Unavailable bool' to the Result struct, or returning a sentinel error. Whichever approach is chosen, it must be backwards-compatible with the existing Updater interface contract.
- Apply() should similarly indicate that updates cannot be performed, without returning an error that would be confused with a transient failure.
- Update the package-level doc comment to reflect the new behavior.
- Keep the stubUpdater's checkFn override mechanism intact for test flexibility.

The choice between a Result field vs. sentinel error is a design decision worth capturing in an ADR after implementation.
