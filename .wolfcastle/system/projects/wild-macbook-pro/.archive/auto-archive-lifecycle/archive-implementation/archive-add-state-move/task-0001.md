# Extract shared archive move logic

The daemon's archiveNode method contains the full move logic (markdown rollup + directory relocation + index mutation), but it lives on the Daemon struct and depends on daemon fields (d.Store, d.Config, d.Clock, d.branch, d.WolfcastleDir). Extract the core move logic into a function or service that both the daemon and the CLI can call. Options: (1) a standalone function in internal/archive/ that takes explicit parameters, (2) a thin service struct. The daemon's archiveNode then delegates to this shared function. The key constraint: the CLI uses app.State (a state.Store) and app.Config, while the daemon uses d.Store and d.Config, but they share the same interfaces. Choose the approach that minimizes code duplication while keeping the daemon and CLI decoupled.
