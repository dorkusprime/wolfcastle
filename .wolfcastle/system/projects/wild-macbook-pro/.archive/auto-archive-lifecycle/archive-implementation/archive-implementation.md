# Archive Implementation

Implements the core auto-archive system as defined by the archive service spec. Covers four concerns: (1) Config types and defaults: add ArchiveConfig to the config tree with auto_archive_delay_hours and auto_archive_enabled fields, wire defaults into base config. (2) State type extensions: add completion timestamp tracking to NodeState/RootIndex, add archived status or flag to index entries so archived nodes are distinguishable from active ones. (3) Core archive service: implement move (relocate node state from active tree to archive storage, update RootIndex), restore (reverse the move, re-integrate into active tree), and delete (permanently remove archived state). (4) CLI commands: wolfcastle archive restore --node <addr>, wolfcastle archive delete --node <addr>, update archive add to also move node state (not just generate markdown). All new and modified code must maintain >90% test coverage.
