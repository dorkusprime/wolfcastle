# Write config show command spec

Write a spec for the 'wolfcastle config show' command. The spec should define:

- Output format: the resolved config printed as indented JSON to stdout (matching the structure in types.go). In --json mode, wrap in the standard {ok, action, data} envelope.
- Flag: --tier <base|custom|local> to display a single tier's raw content instead of the merged result. Omitting --tier shows the fully merged config (the default and most common use case).
- Flag: --raw to suppress the defaults layer and show only what the tier files actually contain (useful for debugging what a specific tier overrides).
- Behavior: the command must work after 'wolfcastle init' (requires a valid .wolfcastle directory). It calls config.Load() for the merged view and reads individual tier files for --tier mode.
- No model invocation required. Pure config read.
- Error handling: if a tier file is missing (e.g., no local/config.json), print {} for that tier rather than erroring.

Place the spec at docs/specs/ with the standard timestamped filename.
