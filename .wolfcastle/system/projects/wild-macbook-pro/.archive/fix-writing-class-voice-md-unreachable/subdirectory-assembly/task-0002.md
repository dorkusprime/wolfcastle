# Add tests for subdirectory assembly behavior

Add test cases to class_repository_test.go covering the new subdirectory assembly behavior:

1. TestClassRepository_Resolve_AppendsSubdirectoryAssets: Set up a class with key.md and key/extra.md. Verify Resolve returns both contents concatenated.
2. TestClassRepository_Resolve_NoSubdirectory: Existing behavior unchanged when no subdirectory exists. Use an existing simple-key test as baseline.
3. TestClassRepository_Resolve_SubdirectoryWithFallback: When a hierarchical key falls back to parent.md, verify that parent/*.md subdirectory assets are also appended.
4. TestClassRepository_Resolve_MultipleSubdirectoryFiles: Multiple .md files in the subdirectory should all be appended, in sorted order.
5. TestClassRepository_Resolve_SubdirectoryScanErrorIgnored: If the subdirectory scan returns an error (not just empty), it should be treated as no-op rather than failing the overall resolve.

Use the existing testutil.NewEnvironment pattern. The WithPrompt helper will need to support subdirectory paths like 'classes/writing/voice.md'.
