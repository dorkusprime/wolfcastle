# Implement subdirectory asset assembly in ClassRepository.Resolve

After ClassRepository.Resolve successfully resolves the main key.md file, it should also check for subdirectory assets at prompts/classes/{key}/. Use PromptRepository.ListFragments("prompts/classes/"+key, nil, nil) to collect any .md files in the subdirectory across all tiers. If fragments are found, concatenate them (separated by newlines) and append to the main file content before returning.

Key design points:
- The subdirectory scan happens AFTER the main file resolves successfully. If the main file doesn't exist, fallback behavior is unchanged.
- Missing subdirectory is not an error; it just means no extra content is appended.
- Subdirectory files are sorted lexicographically (ListFragments already handles this) for deterministic output.
- The assembly applies to both exact-key resolution and parent-key fallback. If key 'typescript/react' falls back to 'typescript.md', subdirectory assets from 'typescript/' should still be appended.
- Thread safety: no new state; ListFragments reads through the existing PromptRepository which is goroutine-safe.

Implementation location: internal/pipeline/class_repository.go, Resolve method (lines 48-78).
Test location: internal/pipeline/class_repository_test.go.
