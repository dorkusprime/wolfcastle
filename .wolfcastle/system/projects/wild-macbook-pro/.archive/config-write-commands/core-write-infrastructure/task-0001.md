# Implement dot-notation path utilities

Create internal/config/path.go with functions for navigating and mutating nested map[string]any structures using dot-delimited keys.

Required functions:

- SetPath(m map[string]any, path string, value any) error
  Splits path on dots, walks/creates intermediate maps, sets the final key.
  Returns error if an intermediate value exists but is not a map.

- DeletePath(m map[string]any, path string) error
  Sets the value to nil at the path (null deletion semantics for DeepMerge).
  Returns error if intermediate path segment is not a map.

- GetPath(m map[string]any, path string) (any, bool)
  Walks the path and returns the value if found.

- ParsePath(path string) ([]string, error)
  Splits on dots, validates no empty segments, no trailing dots, no array indexing syntax.

Edge cases:
- Single-segment paths (top-level keys like 'version')
- Creating intermediate maps when they don't exist (for SetPath)
- Path segments that happen to contain map keys in nested structures (e.g., 'models.fast.command' where 'fast' is a map key)

No array indexing support — return a clear error if brackets are detected in a path segment.
