# Implement path utilities tests

Create internal/config/path_test.go with comprehensive tests for all path utility functions.

Test cases should cover:
- ParsePath: valid paths, empty segments, trailing dots, bracket detection
- SetPath: top-level set, nested set, creating intermediates, overwriting existing values, error on non-map intermediate
- DeletePath: setting nil at path, nested deletion, error on non-map intermediate
- GetPath: existing values, missing values, nested navigation
- Map key paths within nested maps (e.g., pipeline.stages.audit.enabled)
