# Add default task_classes entries to config.Defaults()

Add a TaskClasses map to the return value of config.Defaults() in internal/config/config.go. Include an entry for EVERY class that has a corresponding prompt file authored in Phase 2.

The full list from the spec (verify each has a prompt file before adding):

Language classes: coding/python, coding/javascript, coding/typescript, coding/java, coding/csharp, coding/go, coding/rust, coding/cpp, coding/c, coding/ruby, coding/php, coding/swift, coding/kotlin, coding/scala, coding/shell, coding/sql, coding/r, coding/lua, coding/elixir, coding/haskell, coding/dart

Framework classes: coding/typescript/react, coding/typescript/vue, coding/typescript/angular, coding/typescript/nextjs, coding/typescript/svelte, coding/javascript/react, coding/javascript/node, coding/python/django, coding/python/fastapi, coding/python/flask, coding/ruby/rails, coding/ruby/sinatra, coding/java/spring, coding/csharp/dotnet, coding/php/laravel, coding/php/symfony, coding/kotlin/android, coding/swift/ios, coding/dart/flutter, coding/elixir/phoenix, coding/rust/actix, coding/rust/tokio

Non-language classes: architecture, research, writing, design, devops, data, security, testing, audit

Each entry needs a ClassDef with a Description field matching the spec tables. The Model field should be omitted (empty) for most classes. The research class should have Model: "light" as shown in the spec example.

The class keys must match the prompt file paths under prompts/classes/ so that ClassRepository.Resolve can find them.
