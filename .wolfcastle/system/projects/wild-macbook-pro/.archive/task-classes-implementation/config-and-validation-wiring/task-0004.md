# Auto-assign audit class at claim time

Update the TaskClaim function in internal/state/mutations.go to auto-assign Class "audit" to tasks that have IsAudit == true and an empty Class field.

Current TaskClaim (mutations.go) transitions a task from not_started to in_progress. Add logic:
1. After the state transition succeeds, check if task.IsAudit is true and task.Class is empty
2. If both conditions hold, set task.Class = "audit"
3. The IsAudit field remains the authoritative marker for audit identity; Class is purely for prompt routing

This ensures audit tasks automatically get the audit behavioral prompt without requiring --class audit on every audit task creation.

Add tests in mutations_test.go (or the appropriate test file) verifying:
- Audit task with empty class gets Class = "audit" after claim
- Audit task with existing class is NOT overwritten
- Non-audit task class is not modified by claim
