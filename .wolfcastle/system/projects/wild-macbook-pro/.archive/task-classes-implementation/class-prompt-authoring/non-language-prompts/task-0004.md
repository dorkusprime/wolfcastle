# Author design.md, devops.md, data.md, and security.md

Create four non-language class prompts:

1. internal/project/templates/prompts/classes/design.md — UI/UX design discipline. 30-60 lines. Covers: user goal identification, interaction sequence thinking, edge state handling (empty, error, loading, overflow), accessibility considerations, responsive design awareness, design system consistency. Research current design system practices (Material Design 3, Apple HIG).

2. internal/project/templates/prompts/classes/devops.md — Infrastructure and CI/CD discipline. 30-60 lines. Covers: Dockerfile best practices (multi-stage builds, layer caching), GitHub Actions patterns, Terraform conventions, deployment safety (canary, blue-green awareness), secrets management, monitoring and alerting, infrastructure as code principles. Research current DevOps tooling and practices.

3. internal/project/templates/prompts/classes/data.md — Data engineering and analysis discipline. 30-60 lines. Covers: schema design principles, pipeline idempotency, data validation at boundaries, visualization clarity, SQL style, ETL vs ELT awareness, data quality checks, handling missing/null data. Research current data engineering practices.

4. internal/project/templates/prompts/classes/security.md — Security review and hardening discipline. 30-60 lines. Covers: OWASP Top 10 awareness, threat modeling approach, dependency auditing, secrets detection, input validation patterns, authentication/authorization review, security header configuration, vulnerability disclosure. Research current OWASP guidelines and security best practices.

ZERO wolfcastle-specific content in any file.
