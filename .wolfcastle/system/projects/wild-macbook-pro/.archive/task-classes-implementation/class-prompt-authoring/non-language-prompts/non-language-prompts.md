# Non-Language Prompts

Author behavioral prompts for all 9 non-language discipline classes. Each file goes directly under internal/project/templates/prompts/classes/ (e.g., classes/writing.md, classes/audit.md).

Classes: architecture, research, writing, design, devops, data, security, testing, audit.

Each prompt is 30-60 lines covering discipline-specific approach, quality standards, common pitfalls, and deliverable expectations. The writing class (writing.md) must reference writing/voice.md for voice guidance. The audit class must emphasize read-only review and gap recording.

Also create writing/voice.md as a separate file so users can override voice independently.

ZERO wolfcastle content in any prompt. Researched and current.
