# Add IsAlive callback to FollowReader poll loop

Extend FollowReader in internal/logrender/reader.go to accept an optional 'alive check' function (func() bool). When provided, the poll loop should call it periodically (every N cycles, not every tick, to avoid hammering the PID file) and return if it reports false. Design options: (1) Add the callback as a field on FollowReader with a setter method like SetAliveCheck(fn func() bool). (2) Pass it as a parameter to NewFollowReader. Option 1 is preferable because it keeps the constructor signature stable and the callback is truly optional. In follow.go, wire it up: reader.SetAliveCheck(app.Daemon.IsAlive). The check frequency should be reasonable, something like every 5 seconds (25 cycles at 200ms interval). After the alive check returns false, do one final cycle to drain any remaining buffered lines before closing the channel.
