## Summary

Added daemon exit detection to FollowReader via an optional alive-check callback. The poll loop invokes the callback every 25 cycles (~5s at the 200ms default) and, when it returns false, performs one final drain cycle before closing the record channel. The log command wires app.Daemon.IsAlive through this mechanism for both structured and JSON follow modes. A separate command-level guard downgrades explicit --follow to replay when the daemon is already stopped at startup, preventing the hang that motivated this work. Three reader-level tests and three command-level tests cover the new behavior. Audit passed with no findings requiring remediation.
