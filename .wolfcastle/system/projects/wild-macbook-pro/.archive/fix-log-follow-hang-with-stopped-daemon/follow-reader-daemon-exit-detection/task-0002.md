# Wire alive check in follow.go

In cmd/daemon/follow.go, after creating the FollowReader (line 124), call reader.SetAliveCheck(app.Daemon.IsAlive) to connect the daemon status to the reader's exit detection. This requires threading the daemon repository (or just the IsAlive method) through to runLog. Currently runLog takes (ctx, logDir, session, mode, follow) but has no access to the daemon. Options: (1) Add a func() bool parameter to runLog for the alive check. (2) Pass *daemon.DaemonRepository. Option 1 is cleaner since runLog shouldn't know about the daemon package. The followJSON path (line 206) also needs the same treatment.
