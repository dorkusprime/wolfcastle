# Wire wolfcastle start foreground to InterleavedRenderer

When wolfcastle start runs in foreground mode (no -d flag), it currently relies on Logger.writeConsole() to produce human-readable output on stderr. Instead, adopt the same pattern used by cmd/execute.go and cmd/intake.go:

1. Set Logger.Console to io.Discard (disable writeConsole)
2. Spawn a FollowReader goroutine that tails the log directory
3. Feed records through NewInterleavedRenderer(os.Stdout)
4. Cancel the renderer context when the daemon loop exits

Reference: cmd/execute.go lines 62-86 shows the pattern. The start command is in cmd/daemon/start.go around line 147 (foreground path).

This ensures the same rendering code path is used by wolfcastle start, wolfcastle execute, wolfcastle intake, and wolfcastle log --interleaved.
