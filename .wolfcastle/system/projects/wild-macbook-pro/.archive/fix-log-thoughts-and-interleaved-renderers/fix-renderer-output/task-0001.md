# Reproduce and diagnose raw NDJSON output

Run wolfcastle log --thoughts and wolfcastle log --interleaved against an existing session and confirm both dump raw NDJSON instead of formatted text. Then trace the pipeline to find where formatting is lost. Key files to investigate:

- cmd/daemon/follow.go: runLog() dispatches to renderers at lines 136-163. Verify the correct renderer is instantiated and Render() is called.
- internal/logrender/reader.go: ReplayReader.Records() and FollowReader.Records() parse NDJSON and emit Record structs. Verify records are being parsed (not passed as raw strings).
- internal/logrender/thoughts.go: ThoughtsRenderer.Render() filters for type==assistant and prints r.Text. Verify this path executes.
- internal/logrender/interleaved.go: InterleavedRenderer.Render() handles stage_start, stage_complete, assistant, etc. Verify this path executes.

Possible failure modes: (a) renderer is constructed but never receives records from the channel, (b) records arrive but fail the type checks, (c) a fallback path dumps raw JSON, (d) the wrong output mode is selected, (e) the reader silently fails to parse and some other codepath writes stdout.
