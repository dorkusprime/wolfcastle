# Cover iteration and autoCommit paths

Two iteration-phase functions need coverage attention:

runIteration (84.6%, iteration.go:23):
- Pending-filing skip path (task already filed by another iteration)
- Invoke error with non-context error
- Failure escalation paths: decomposition trigger, hard failure cap reached
- Task complete with deliverable check failure
- Auto-commit trigger conditions

autoCommitPartialWork (76.5%, iteration.go:414):
- Git HEAD unchanged (no progress) path
- Commit command failure
- Empty diff (nothing to commit) scenario
- Multiple files changed in single commit

These functions interact closely with the state store, so tests need realistic node state setup with tasks in various lifecycle stages.
