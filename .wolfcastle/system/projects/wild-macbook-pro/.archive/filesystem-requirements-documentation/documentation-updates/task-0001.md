# Add filesystem requirements section to README.md

Add a section to README.md documenting that the .wolfcastle/ directory must reside on a local filesystem (ext4, APFS, etc.), not on NFS or other network-mounted filesystems. Explain that Wolfcastle uses flock(2) advisory locking to protect state from concurrent access, and that flock(2) is not honored by NFS or most network filesystems. State the specific risk: concurrent access from network-mounted directories can corrupt state silently, with no error reported to the user. Place this section near the configuration or collaboration sections where users would naturally look for deployment constraints. Keep it concise: a short paragraph with the requirement, the reason, and the consequence of violating it.
