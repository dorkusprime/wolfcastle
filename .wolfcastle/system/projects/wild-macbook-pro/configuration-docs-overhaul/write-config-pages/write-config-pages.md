# Write Config Pages

Author the four configuration documentation pages: config-quickstart.md (five-minute getting-started), configuration.md (rewritten how-it-works guide covering tiers, merge semantics, common customizations, CLI commands), config-reference.md (every config field by section with type/default/description/example), and task-classes.md (dedicated class system page with language/framework tables, custom classes, fallback behavior). Each page is a separate leaf. Cross-references between pages should be wired as each page is written.
