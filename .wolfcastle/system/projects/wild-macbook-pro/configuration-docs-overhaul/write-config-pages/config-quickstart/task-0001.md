# Write config-quickstart.md

Create docs/humans/config-quickstart.md as a five-minute getting-started page. Four recipes, each 3-5 lines of prose plus one code block. No theory.

1. Change Your Model — show the exact JSON to put in system/custom/config.json to switch the heavy model to a different provider/model. Then show the wolfcastle config set CLI alternative for the same change. Keep it concrete: here is the JSON, here is where to put it, done.

2. Add a Custom Stage — show how to add a new stage (e.g., spec-review) to the pipeline. Include both the stage definition in the stages dict and adding it to stage_order. Show the JSON snippet for custom/config.json.

3. Override a Prompt — show where to place a custom prompt file in system/custom/prompts/ to override a base prompt. Explain that same-named files in higher tiers replace lower-tier versions. One example: overriding execute.md.

4. Set a Task Class — brief example with --class coding/typescript on wolfcastle task add. Show what it does (selects a behavioral prompt). Link to the full Task Classes page (task-classes.md) for the complete class list and custom class creation.

Open with a one-line intro: 'Four common configuration tasks, each under a minute.' Close with links: 'For how configuration works under the hood, see the [Configuration Guide](configuration.md). For every available field, see the [Config Reference](config-reference.md).'

Keep it tight. This page exists for people who want to change something RIGHT NOW, not understand the system.
