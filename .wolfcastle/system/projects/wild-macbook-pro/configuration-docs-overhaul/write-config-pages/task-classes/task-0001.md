# Write task-classes.md

Create docs/humans/task-classes.md covering the full class system.

Primary sources: the task classes spec at .wolfcastle/docs/specs/2026-03-15T00-04Z-task-classes.md, the ClassDef type in internal/config/types.go, and the ClassRepository in internal/pipeline/class_repository.go.

Page structure:
1. What Classes Are — behavioral prompts that shape how the agent approaches work. Two layers: universal.md (always applied) + class-specific prompt. Classes don't restrict tools, don't change markers, don't gate capabilities.

2. Using a Class — show 'wolfcastle task add --class coding/typescript' for languages, '--class coding/typescript/react' for frameworks. Explain the fallback: coding/typescript/react tries the framework prompt first, falls back to coding/typescript if the framework prompt doesn't exist.

3. Built-in Language Classes — full table with class key and one-line description. Pull from Defaults() in config.go. Group by category: systems (go, rust, c, cpp), JVM/.NET (java, kotlin, scala, csharp), web/scripting (javascript, typescript, python, ruby, php, lua, elixir, dart), data/specialized (r, sql, haskell, swift, shell).

4. Built-in Framework Classes — full table grouped by parent language. Pull from Defaults(). Show class key and what the framework prompt covers.

5. Non-language Classes — table of discipline classes (architecture, research, writing, design, devops, data, security, testing, audit) with descriptions from Defaults().

6. Creating a Custom Class — step-by-step: add ClassDef to system/custom/config.json, create .md file in system/custom/prompts/classes/, verify with 'wolfcastle config show --section task_classes'. Show a complete example (e.g., coding/go-internal with team conventions).

7. Overriding a Built-in Class — place a file with the same path in system/custom/prompts/classes/ to replace the base prompt. Only describe coding conventions; no wolfcastle internals needed.

8. Creating a Custom Framework — add subdirectory under parent language, drop .md file, add config entry. Show fallback behavior.

9. The Universal Prompt — what universal.md contains, that it applies to every task, how to override it.

Cross-references: link to Configuration Guide for tier mechanics, link to Config Reference for the task_classes config section.
