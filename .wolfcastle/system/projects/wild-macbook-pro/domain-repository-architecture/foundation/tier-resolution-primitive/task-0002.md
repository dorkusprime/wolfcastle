# Write comprehensive tests for tierfs

Create internal/tierfs/tierfs_test.go with table-driven tests covering:

Resolve:
- Returns content from highest-priority tier (local > custom > base)
- Falls through to lower tiers when higher tiers lack the file
- Returns base tier content when only base has the file
- Returns wrapped os.ErrNotExist when no tier has the file
- Handles subdirectory paths (e.g., "prompts/execute.md")

ResolveAll:
- Collects files across all tiers
- Higher tier files override same-named files from lower tiers
- Only includes .md files (skips directories, non-.md files)
- Returns empty map when subdirectory doesn't exist in any tier
- Handles tiers where the subdirectory is missing (some tiers may not have the subdir)

WriteBase:
- Writes file to base tier directory
- Creates parent directories as needed
- Overwrites existing file

BasePath:
- Returns correct absolute path to base tier subdirectory

TierDirs:
- Returns all three tier directories in order (base, custom, local)

Each test should create a temp directory structure using t.TempDir(), populate it with the needed tier files, construct an FS, and verify behavior. Keep tests self-contained — no dependency on testutil.Environment (that package depends on tierfs, not the reverse).
