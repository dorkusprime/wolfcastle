# Implement With* helper methods

Add the fluent With* methods to Environment in internal/testutil/environment.go (or a separate file if cleaner):

WithConfig(overrides map[string]any) *Environment:
- Deep-merges overrides into the custom tier config file (system/custom/config.json)
- Reads existing custom config, applies DeepMerge from internal/config, writes back
- Returns the same *Environment for chaining

WithProject(name string, root NodeSpec) *Environment:
- Creates a project in the state tree using the NodeSpec structure
- Writes node state files and updates the root index
- Handles recursive NodeSpec children: orchestrators create child directories and child state files
- Each leaf's Tasks list creates task entries with auto-generated descriptions ("Task: <id>")
- Each node gets an audit task automatically (matching current SetupTree behavior)
- Returns the same *Environment for chaining

WithPrompt(relPath string, content string) *Environment:
- Writes a prompt file to system/base/prompts/<relPath>
- Uses the Environment's tierfs.FS.WriteBase("prompts/" + relPath, []byte(content))
- Returns the same *Environment for chaining

WithRule(name string, content string) *Environment:
- Writes a rule fragment to system/base/rules/<name>
- Uses the Environment's tierfs.FS.WriteBase("rules/" + name, []byte(content))
- Returns the same *Environment for chaining

All With* methods should call t.Fatal on error (the Environment holds the *testing.T). This keeps test code clean — no error checking at every setup step.
