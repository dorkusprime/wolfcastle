# Replace buildIterationContext callers with ContextBuilder

Update all call sites that currently use BuildIterationContext(), BuildIterationContextWithDir(), or BuildIterationContextFull() to use ContextBuilder.Build() instead.

Expected callers are in the daemon pipeline and possibly command handlers. Search for all references to these functions. Each caller will need access to a ContextBuilder instance, which means either passing it as a parameter or constructing it from available repositories.

After migrating all callers, remove or deprecate the old standalone functions in context.go. Keep the file if other pipeline utilities remain in it; remove it entirely if it becomes empty.

Verify the build passes and tests pass after the switch.
