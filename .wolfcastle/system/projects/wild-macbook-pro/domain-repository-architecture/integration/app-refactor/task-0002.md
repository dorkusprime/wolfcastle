# Update command handlers for new App fields

Update all command handler files in cmd/ that reference the old App fields (WolfcastleDir, Cfg, Resolver) to use the new repository-based fields.

Common patterns to replace:
- app.Cfg.SomeField → load from app.Config.Load() or cache the loaded config
- app.WolfcastleDir → use repository methods instead (repositories already know their root)
- app.Resolver.ProjectsDir() → app.Identity.ProjectsDir(root) or app.State methods
- app.Resolver.NodeDir(addr) → app.State methods
- app.RequireResolver() → app.RequireIdentity()

Search all files in cmd/ for references to these fields. Update each one. Some commands (init, update) don't need identity and should skip RequireIdentity(). Most commands (project, task, daemon, audit, navigate, status) need identity.

Verify the build passes after each file is updated.
