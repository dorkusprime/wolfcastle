# Tier-Based Repositories

Implement ConfigRepository and PromptRepository, both of which depend on tierfs for three-tier file resolution. ConfigRepository (internal/config) owns the tier-based config merge: Load returns a merged *Config from defaults plus each existing tier, WriteBase takes a complete *Config snapshot, WriteCustom/WriteLocal take partial map[string]any overlays. PromptRepository (internal/pipeline) owns prompt template resolution across tiers: Resolve expands Go text/template prompts by short name, ResolveRaw returns raw content by category/name, ListFragments collects all fragments in a category with tier-override semantics, WriteBase and WriteAllBase write to the base tier. Both accept tierfs.Resolver in their constructors for testability and follow the NewXxx(wolfcastleRoot) / NewXxxWithTiers(tiers) dual-constructor pattern from the spec.
