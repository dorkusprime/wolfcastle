# Implement Identity type and extraction functions

Create internal/config/identity.go with:

- Identity struct: User string, Machine string, Namespace string (computed as User + "-" + Machine).
- IdentityFromConfig(cfg *Config) (*Identity, error): extracts identity from a loaded config. Returns error if cfg.Identity is nil or if User/Machine are empty. Sets Namespace to User + "-" + Machine.
- DetectIdentity() *Identity: reads username from os/user.Current() and hostname from os.Hostname(). Falls back to "unknown" for either if the system call fails. Sets Namespace from the detected values.
- (id *Identity) ProjectsDir(wolfcastleRoot string) string: returns filepath.Join(wolfcastleRoot, "system", "projects", id.Namespace).

This mirrors the logic currently in tree.ResolveNamespace and project.detectIdentity, but as a first-class type. The existing IdentityConfig struct in types.go remains as the config-tier representation; Identity is the domain type derived from it.
