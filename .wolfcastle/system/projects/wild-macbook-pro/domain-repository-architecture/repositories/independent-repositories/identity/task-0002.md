# Test Identity type

Create internal/config/identity_test.go with:

1. TestIdentityFromConfig_Valid: Provide a Config with Identity.User and Identity.Machine set. Verify the returned Identity has correct User, Machine, and Namespace (User + "-" + Machine).
2. TestIdentityFromConfig_NilIdentity: Pass a Config with nil Identity, verify error.
3. TestIdentityFromConfig_EmptyUser: Identity with empty User, verify error.
4. TestIdentityFromConfig_EmptyMachine: Identity with empty Machine, verify error.
5. TestDetectIdentity: Call DetectIdentity, verify it returns a non-nil Identity with non-empty User, Machine, and Namespace. (We can't assert exact values since they depend on the host, but we can verify the contract.)
6. TestIdentity_ProjectsDir: Create an Identity with known values, call ProjectsDir with a temp root, verify the returned path matches the expected structure (root/system/projects/namespace).

These are unit tests — no testutil.Environment needed since Identity has no filesystem dependencies beyond ProjectsDir path construction.
