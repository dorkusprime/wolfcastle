# Implement git.Provider interface and git.Service struct

Create internal/git/git.go with:

- Provider interface: CurrentBranch() (string, error), HEAD() string, HasProgress(sinceCommit string) bool, IsRepo() bool, IsDirty(excludePaths ...string) bool, CreateWorktree(path, branch string) error, RemoveWorktree(path string) error.
- Service struct with repoDir string field.
- NewService(repoDir string) *Service constructor.

Implement each method by shelling out to git:

- CurrentBranch: git rev-parse --abbrev-ref HEAD, with fallback for empty repos (see daemon/branch.go for the three-step logic: rev-parse, then git-dir check, then symbolic-ref).
- HEAD: git rev-parse HEAD, returns empty string on error.
- IsRepo: git rev-parse --is-inside-work-tree succeeds.
- IsDirty: git status --porcelain, filtering out lines matching excludePaths. Exclude paths by prefix-matching the file path column in porcelain output.
- HasProgress: HEAD() != sinceCommit || IsDirty(".wolfcastle/"). The .wolfcastle/ exclusion is baked into the HasProgress call.
- CreateWorktree: git worktree add <path> -b <branch>.
- RemoveWorktree: git worktree remove <path>.

All commands run with cmd.Dir = repoDir.
