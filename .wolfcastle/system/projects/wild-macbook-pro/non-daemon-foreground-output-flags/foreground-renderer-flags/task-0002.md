# Add tests for foreground output mode flag wiring

Write table-driven tests for the output mode flag parsing on the start command. The test should:
1. Create a startCmd via newStartCmd (or the registered command) with various flag combinations.
2. Parse the flags using cmd.ParseFlags or cmd.SetArgs and verify the mode variable is set correctly.
3. Test last-wins semantics: passing multiple mode flags should result in the last one taking effect.

If the mode variable is scoped inside newStartCmd, the test may need to verify behavior indirectly (e.g., by checking that the flags are registered on the command, or by testing a helper function that maps mode to renderer). Follow the same testing approach used for the log command's mode flags if tests exist in follow_test.go.
