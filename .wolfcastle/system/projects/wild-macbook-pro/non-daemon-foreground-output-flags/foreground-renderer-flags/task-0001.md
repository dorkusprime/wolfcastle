# Add output mode flags to start command and wire renderer dispatch

In register.go, call registerModeFlags(startCmd, &mode) where mode is a package-level or closure-scoped outputMode variable defaulting to modeSummary. The mode variable must be accessible from start.go's RunE.

In start.go, replace the hardcoded InterleavedRenderer block (lines 154-165) with a switch on mode:
- modeSummary: logrender.NewSummaryRenderer(os.Stdout).Follow(ctx, records)
- modeThoughts: logrender.NewThoughtsRenderer(os.Stdout).Render(ctx, records)
- modeInterleaved: logrender.NewInterleavedRenderer(os.Stdout).Render(ctx, records)
- modeJSON: marshal each record from the channel to JSON and write to stdout

The mode variable needs to flow from flag registration in register.go to the RunE closure in start.go. Options: (a) make newStartCmd accept a *outputMode parameter, (b) use a closure in register.go, or (c) register flags inside newStartCmd. Option (c) matches how newLogCmd handles it (mode declared at the top of newLogCmd, registerModeFlags called on cmd inside newLogCmd). Follow the same pattern: declare mode inside newStartCmd and call registerModeFlags there.

When -d is passed, the mode flags should have no effect (background mode redirects to daemon.log and never renders to stdout).
