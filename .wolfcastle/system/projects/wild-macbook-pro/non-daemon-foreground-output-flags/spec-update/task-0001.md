# Update Non-Daemon Mode section in log-command-design spec

The existing log-command-design spec (2026-03-21T18-00Z) has a Non-Daemon Mode section (lines 129-133) that says non-daemon output 'should match the interleaved format.' This contradicts the new requirement: foreground mode should default to summary and accept the same flags as wolfcastle log. Update that section to:
1. State that the default renderer is summary (one line per completed stage).
2. Document that --thoughts, --interleaved, and --json flags are accepted with the same last-wins semantics as wolfcastle log.
3. Keep the architectural point that rendering is done by a goroutine tailing the NDJSON log file, sharing renderers with wolfcastle log.
4. Remove or correct any statement that says interleaved is the default for non-daemon mode.
