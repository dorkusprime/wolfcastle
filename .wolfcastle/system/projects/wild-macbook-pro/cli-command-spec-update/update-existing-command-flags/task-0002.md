# Add missing flags to wolfcastle project create spec section

The project create command in cmd/project/create.go registers 3 flags not documented in the spec. Update the spec's project create section (line ~1381) to include:

1. --type (string): Node type, either 'leaf' or 'orchestrator'. Defaults to leaf. The current spec says the node is always an orchestrator, which is incorrect; the --type flag controls this.
2. --description (string): Project description text (written to the project .md file)
3. --scope (string): Scope description for orchestrator nodes (used by the planning agent)

Update the Arguments and Flags table, Behavior section (explain how --type affects node creation, how --description populates the .md file, how --scope is stored), Output section (include type in output), and fix the Description paragraph that currently says 'The new node is an orchestrator' to reflect that --type controls this.
