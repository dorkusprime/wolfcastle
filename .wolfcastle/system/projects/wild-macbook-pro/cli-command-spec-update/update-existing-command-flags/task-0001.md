# Add missing flags to wolfcastle task add spec section

The task add command in cmd/task/add.go registers 10 flags not documented in the spec. Update the spec's task add section (line ~777) to include:

1. --body (string): Detailed task description/body
2. --stdin (bool): Read task body from stdin
3. --deliverable (string slice, repeatable): Expected output file
4. --type (string): Task type enum (discovery, spec, adr, implementation, integration, cleanup)
5. --class (string): Task class override (e.g., lang-go)
6. --constraint (string slice, repeatable): What not to do
7. --acceptance (string slice, repeatable): Acceptance criterion
8. --reference (string slice, repeatable): Reference material path
9. --integration (string): How this task connects to other work
10. --parent (string): Parent task ID for hierarchical decomposition

Update the Arguments and Flags table, Behavior section (explain how each flag is stored in state.json), Output section (show deliverables in output), and Examples section. The Synopsis should also be updated to show available flags.
