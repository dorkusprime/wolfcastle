# Add ReadTier and ApplyMutation to ConfigRepository

Extend internal/config/repository.go with methods that support the read-modify-write-validate cycle for config write commands.

Add these methods to ConfigRepository:

- ReadTier(tier string) (map[string]any, error)
  Reads and parses a single tier's config.json overlay. Returns empty map if file doesn't exist. Accepts 'custom' or 'local' (reject 'base' with a clear error — base is written only via WriteBase with a full Config).

- WriteTier(tier string, overlay map[string]any) error
  Writes an overlay to the specified tier. Dispatches to WriteCustom or WriteLocal based on tier name. Rejects 'base'.

- ApplyMutation(tier string, mutate func(overlay map[string]any) error) error
  High-level helper: reads the tier overlay, calls mutate to modify it in-place, writes the overlay, then calls Load() to validate the merged result. If Load() returns a validation error, restores the original tier file contents and returns the error. This provides atomic write-or-rollback semantics.

Note: The cmd/config/show.go already has a readTierFile helper. The new ReadTier method on ConfigRepository should provide similar functionality but as a proper method on the repository, allowing the show.go helper to optionally be refactored later. Do not break show.go — just add the new methods.
