# Write comprehensive tests for all config write commands

Create cmd/config/write_test.go (or set_test.go, unset_test.go, etc.) with tests for all four write commands.

Follow the testing patterns established in show_test.go:
- Use testEnv setup with a real ConfigRepository
- Test through the cobra command execution (cmd.Execute)

Test cases for config set:
- Set a top-level scalar (version)
- Set a nested value (daemon.poll_interval_seconds)
- Set a value in a map-keyed structure (models.fast.command)
- Set with JSON value (object, array, number, boolean)
- Set with bare string value
- Set to custom tier
- Set that fails validation (e.g., daemon.poll_interval_seconds to negative) — verify rollback
- Invalid key (empty segment, trailing dot)
- Invalid tier name

Test cases for config unset:
- Unset an existing key
- Unset a nested key
- Unset a non-existent key (should succeed silently — writing nil is idempotent)

Test cases for config append:
- Append to existing array
- Append to non-existent key (creates single-element array)
- Append to non-array value (error)

Test cases for config remove:
- Remove from existing array
- Remove value not in array (error)
- Remove from non-array (error)

All tests should verify both human and JSON output modes.
