# Add filesystem requirements to SECURITY.md

Add a section to SECURITY.md documenting the filesystem locality requirement as a security consideration. The current SECURITY.md already lists 'State file corruption bypassing flock mechanism' as an in-scope vulnerability. Expand on this by adding a dedicated section (e.g., 'Filesystem Requirements') that explains: (1) .wolfcastle/ must reside on a local filesystem, (2) flock(2) advisory locks are not propagated over NFS or network filesystems, (3) running Wolfcastle against a network-mounted .wolfcastle/ directory means the locking mechanism silently becomes a no-op, allowing concurrent processes to read and write state without mutual exclusion, (4) the result is silent state corruption with no error or warning. Frame this as a deployment constraint that operators must respect.
