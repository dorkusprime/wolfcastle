# Add tests for FollowReader daemon-exit detection

Add tests to internal/logrender/reader_test.go that verify: (1) FollowReader with an alive check that returns false stops polling and closes the channel promptly. (2) FollowReader with an alive check that returns true continues polling normally. (3) FollowReader with no alive check set (nil) polls indefinitely as before (backward compat). (4) The final drain cycle runs before closing, so no lines are lost. Use a test-controlled alive function (a closure over a boolean or atomic) that can be flipped mid-test. Set a short poll interval (10ms) and a tight test timeout to catch hangs.
