# Guard follow mode behind daemon-alive check

In cmd/daemon/follow.go, after the implicit-follow block (lines 89-92), add a second guard that downgrades follow to false when the daemon is not alive. This covers the case where the user explicitly passes --follow but the daemon is stopped. The logic should be: if follow is true but app.Daemon.IsAlive() returns false, set follow back to false. This ensures the command falls through to the replay path (line 142) which reads from session files and exits cleanly. The spec (docs/specs/2026-03-21T18-00Z-log-command-design.md, flags table) says --follow is 'No-op when daemon is stopped.' This is the minimal, correct fix for the invocation-time hang.
