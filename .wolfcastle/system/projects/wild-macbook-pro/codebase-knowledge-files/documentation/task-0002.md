# Add CLI documentation pages for knowledge commands

Update docs/humans/cli.md (or add new pages under docs/humans/cli/ if that directory structure exists) to document the four knowledge commands:

1. wolfcastle knowledge add "<entry>" - append a knowledge entry to the current namespace's file. Document the budget enforcement behavior and error message.
2. wolfcastle knowledge show - display the current namespace's knowledge file
3. wolfcastle knowledge edit - open the knowledge file in $EDITOR
4. wolfcastle knowledge prune - open the knowledge file for pruning, report token count after

Include usage examples, flags, and typical workflows. Follow the format of existing CLI documentation.
