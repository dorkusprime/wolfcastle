# Implement wolfcastle knowledge show, edit, and prune commands

Implement the remaining three knowledge commands:

1. 'wolfcastle knowledge show' - reads and prints the current namespace's knowledge file. Supports --json output. If no knowledge file exists, prints a message indicating no knowledge has been recorded yet.

2. 'wolfcastle knowledge edit' - opens the knowledge file in $EDITOR (or falls back to vi). Creates the file with an empty template if it doesn't exist. Similar to how other edit commands work in the codebase.

3. 'wolfcastle knowledge prune' - opens the knowledge file in $EDITOR for manual pruning. Functionally similar to 'edit' but with a different intent (reducing file size). When run by the daemon's maintenance task, operates non-interactively. After editing, reports the new token count relative to the budget.

All three commands call app.RequireIdentity() and follow existing cobra patterns.
