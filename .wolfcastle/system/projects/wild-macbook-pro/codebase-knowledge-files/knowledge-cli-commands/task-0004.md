# Add integration tests for knowledge CLI commands

Write integration tests for all four knowledge commands:

1. knowledge add: test successful append, test budget overflow error, test file creation on first add
2. knowledge show: test display of existing file, test empty/missing file message, test --json output
3. knowledge edit: test that it launches editor (may need to mock EDITOR)
4. knowledge prune: test token count reporting after edit

Use the existing test patterns from other cmd/ packages (testutil.Environment, etc). Ensure tests cover the budget enforcement behavior: adding an entry that would exceed max_tokens must fail with the specified error message.
