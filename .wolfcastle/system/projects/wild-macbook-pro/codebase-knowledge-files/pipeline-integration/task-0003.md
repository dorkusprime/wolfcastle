# Update execute.md prompt with knowledge file instructions

Add an instruction to the execute.md prompt telling agents to update the knowledge file when they discover something non-obvious about the codebase. Per the spec, the instruction should read:

'If you discover something about this codebase that would be useful to future tasks and is not documented elsewhere (in the README, CONTRIBUTING.md, specs, or ADRs), append it to the codebase knowledge file using wolfcastle knowledge add "<entry>".'

Place this instruction in the appropriate section of execute.md (likely in the Record or Document phase, after the main implementation work). The instruction belongs in execute.md (system mechanics), not in class prompts. Keep it concise; agents should understand when and how to use it from a single paragraph.
