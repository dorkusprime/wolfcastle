# Extend ContextBuilder to inject knowledge file content

Add a new section to ContextBuilder.Build that injects the codebase knowledge file content. Per the spec, this section appears after universal/class guidance (current step 4b) and before prior task AARs (current step 5).

Implementation:
1. ContextBuilder.Build needs access to the wolfcastle directory path and the current namespace to locate the knowledge file. Add these as parameters to Build, or pass them through the config/identity. Check how identity (namespace) flows through the daemon to determine the cleanest approach.
2. Call internal/knowledge.Read(wolfcastleDir, namespace) to get the file content
3. If content is non-empty, emit it under a '## Codebase Knowledge' header
4. If no knowledge file exists or it's empty, omit the section entirely (no empty header)
5. The file is read every iteration, never cached, so updates from one task are immediately visible to the next

Update the ContextBuilder contract spec (2026-03-18T23-11Z-contextbuilder-contract.md) to document the new section in the assembly order.

Update existing ContextBuilder tests to verify knowledge injection in the correct position.
