# Add daemon maintenance task creation for knowledge budget overflow

Add a daemon check that creates a maintenance task when a knowledge file exceeds its token budget. Follow the pattern established by spec_review.go for maintenance task creation.

Implementation:
1. Add a method (e.g., checkKnowledgeBudget) that runs during the daemon's iteration loop
2. Read the knowledge file for the current namespace using internal/knowledge.Read
3. Count tokens using internal/knowledge.TokenCount
4. If token count exceeds the configured knowledge.max_tokens, create a maintenance task on the current node:
   - Title: 'Prune codebase knowledge file'
   - Body: instructions to review the knowledge file, remove stale entries, consolidate related entries, and bring it under budget. Include current token count and budget limit.
   - Insert before the audit task using the state.MoveAuditLast pattern
5. Only create the maintenance task once (don't create duplicates). Check for existing maintenance tasks with a known TaskType or title pattern before creating.
6. Log the maintenance task creation

Add tests covering: maintenance task creation on overflow, no duplicate creation, no task when within budget.
