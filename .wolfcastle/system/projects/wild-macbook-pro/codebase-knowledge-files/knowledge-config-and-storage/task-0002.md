# Create internal/knowledge package with file operations and token counting

Create an internal/knowledge/ package that provides:

1. FilePath(wolfcastleDir, namespace string) string - returns the path to .wolfcastle/docs/knowledge/<namespace>.md
2. Read(wolfcastleDir, namespace string) (string, error) - reads the knowledge file, returns empty string (not error) if file doesn't exist
3. Append(wolfcastleDir, namespace, entry string) error - appends a markdown bullet to the file, creating it and parent directories if needed. Adds '- ' prefix if missing, ensures trailing newline
4. TokenCount(content string) (int, error) - counts tokens in the knowledge file content. Use a simple word-based heuristic (words / 0.75) consistent with how the codebase counts tokens elsewhere, or match the existing token counting approach if one exists
5. CheckBudget(wolfcastleDir, namespace string, maxTokens int, newEntry string) error - checks if appending newEntry would exceed maxTokens. Returns a descriptive error if over budget, nil if within budget

All functions should be pure utilities with no daemon or state dependencies. Test all functions including edge cases: empty file, missing file, file creation, budget boundary.
