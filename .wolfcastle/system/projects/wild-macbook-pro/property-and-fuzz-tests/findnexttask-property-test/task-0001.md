# Define FindNextTask navigation invariants spec

Write a spec defining the invariants that FindNextTask must satisfy for any valid project tree. These invariants will be the assertions in the property-based test.

Invariants to define:
1. If Found is true, the returned task must not be in StatusComplete or StatusBlocked state.
2. If Found is true and non-audit not_started tasks exist in reachable nodes, the returned task must not be an audit task.
3. If Found is true, the returned task must not be a parent task that has children (derived status tasks are not directly actionable).
4. If Found is true, the returned task must not have a not_started non-audit ancestor (child tasks are blocked by their parent).
5. If all reachable tasks are complete, Found must be false with reason all_complete.
6. If all reachable tasks are blocked (and none are complete-able), Found must be false with reason all_blocked.
7. If the tree is empty, Found must be false with reason empty_tree.

Study findActionableTask in internal/state/navigation.go (lines 110-220) and propagation_property_test.go for the random tree generation pattern. The spec should be precise enough that a test author can implement assertions directly from it.
