# Implement scanTerminalMarker fuzz test

Implement a Go fuzz test (func FuzzScanTerminalMarker) for the scanTerminalMarker function in internal/daemon/iteration.go.

Model output is untrusted input, so this function must handle arbitrary byte sequences without panicking.

Key implementation details:
- Use Go's native fuzzing (testing.F with f.Fuzz). No external libraries needed.
- The fuzz target receives a single string argument and calls scanTerminalMarker(input).
- The test should NOT assert specific return values (the function returns a string, and any return value is valid). The only property under test is: does not panic.
- Seed the corpus with realistic examples:
  * Plain text with WOLFCASTLE_COMPLETE on its own line
  * WOLFCASTLE_BLOCKED wrapped in asterisks/backticks
  * WOLFCASTLE_SKIP with a reason suffix
  * Valid JSON stream envelopes: {"type":"assistant","text":"WOLFCASTLE_COMPLETE"}
  * Malformed JSON: truncated, nested, missing closing braces
  * Mixed format: some lines raw text, some lines JSON
  * Empty string, very long strings, strings with null bytes
  * Unicode edge cases (RTL markers, zero-width joiners, combining characters)
  * Lines containing partial marker prefixes (WOLFCASTLE_ without a suffix)

Place the test in internal/daemon/iteration_fuzz_test.go (separate file, since fuzz tests often run independently).

Note: scanTerminalMarker is unexported, so the test must be in package daemon.
