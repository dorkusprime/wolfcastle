# Refactor commitAfterIteration to accept and use task metadata

Change commitAfterIteration's signature to accept task metadata: title (string), class (string), deliverables ([]string), latestBreadcrumb (string), and failureType (string, only relevant for failures). Use the CommitPrefix from gitCfg to build the subject line.

Subject line format:
- Success: '{prefix}: {title}' (or '{title}' if prefix is empty)
- Failure: '{prefix}: {title} (attempt N)' (or '{title} (attempt N)' if prefix is empty)

Body format (multi-line, use git commit -m for subject and a second -m for body):
  Task: {taskID} [{class}]
  Deliverables: {comma-separated deliverables}

  {latest breadcrumb text}

For failures, add a line: Failure: {failureType}

If title is empty, fall back to the current behavior (using taskID in the subject).

Update both call sites in runIteration (lines ~381 and ~402). At both sites, the NodeState (ns) is in scope. Extract the task's Title, Class, Deliverables from ns.Tasks by matching nav.TaskID. Extract the latest breadcrumb from ns.Audit.Breadcrumbs (the last entry, or empty string if none). At the failure site, pass the failureType variable that's already computed.

Reference: internal/state/types.go for Task and Breadcrumb struct definitions.
