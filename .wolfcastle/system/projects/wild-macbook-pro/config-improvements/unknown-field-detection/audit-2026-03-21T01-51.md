## Summary

Implemented unknown-field detection in config unmarshalling using a diff-based round-trip approach: raw JSON is leniently unmarshalled into Config, marshalled back to a map, and the keys are recursively compared to identify any that survived in the original but not the round-trip. Detection runs per-tier before merge, labeling each warning with its source file. Handles top-level, nested, map-typed (models, task_classes), and array-element fields with bracket-indexed paths. Warnings are collected on Config.Warnings and surfaced to stderr during app initialization. The implementation is 71 lines with 318 lines of thorough unit and integration tests covering typos, nested paths, arbitrary map keys, array elements, multi-tier accumulation, clean configs, and invalid JSON. All audits passed with no remediations required.

## Findings

1 total (1 remediated, 0 open)

### Remediated

- **gap-unknown-field-detection-1**: diffKeys does not recurse into JSON arrays, so unknown fields inside slice-of-struct elements (pipeline.stages, validation.commands) are not detected. Low priority since these are rarely user-edited in tier overlays. (fixed by config-improvements/unknown-field-detection)
