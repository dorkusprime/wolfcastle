# Remove duplicate console formatting from Logger

With all foreground modes now using InterleavedRenderer, the Logger's writeConsole() method, consoleMessages map, and consoleSuppress map are dead code. Remove them:

- internal/logging/logger.go: Delete writeConsole() (lines 199-244), consoleMessages (lines 189-194), consoleSuppress (lines 180-185)
- internal/logging/logger.go: Remove the console-mirroring block from Log() (lines 170-173) if Console is always io.Discard in foreground mode. If Logger.Console is still used for background daemon log file output, keep the Log() path but remove the human formatting.
- internal/daemon/daemon.go: Remove PrintHuman calls that duplicate renderer output (line 118 warning about missing class prompts, lines 144-149 self-heal messages). If these messages carry information the renderer wouldn't show (they aren't stage events), consider whether they should become proper NDJSON records that the interleaved renderer can display, or whether they should remain as one-off PrintHuman calls. Use judgment: PrintHuman is acceptable for one-time startup diagnostics that happen before the renderer goroutine is running.
- internal/daemon/iteration.go: Evaluate the PrintHuman at line 196 (superseded block message). Same judgment call.

Be careful not to remove output that serves a purpose the renderer can't fulfill (e.g., early startup messages before log files exist).
