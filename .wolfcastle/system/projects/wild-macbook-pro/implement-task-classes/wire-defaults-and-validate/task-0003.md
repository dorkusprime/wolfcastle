# Add --class validation to task add CLI

In cmd/task/add.go, validate the --class flag value against the config's TaskClasses map when it is non-empty. If the provided class key is not found in cfg.TaskClasses, reject the command with an error that lists the valid class values (sorted alphabetically). The App struct in cmd/cmdutil/app.go already has a Classes field with the loaded ClassRepository, and the config is available. Use Classes.List() to get valid keys for the error message. Empty --class values remain valid (no class assigned). This gives users immediate feedback on typos rather than discovering at execution time that the class prompt doesn't resolve.
