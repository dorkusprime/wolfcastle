# Auto-assign Class 'audit' to audit tasks at claim time

When the daemon claims a task (transitions it from not_started to in_progress), check if the task is an audit task (task.IsAudit is true) and its Class field is empty. If both conditions hold, set task.Class to 'audit' before persisting the state change. This ensures audit tasks always get the audit class behavioral prompt without requiring manual --class assignment. The IsAudit field remains the authoritative marker for audit identity; Class is purely for prompt routing. Find where task claiming happens in the daemon (likely internal/daemon/ or internal/state/) and add the auto-assignment there. Write a test that verifies: claiming an audit task with empty class sets class to 'audit'; claiming an audit task with an existing class does not overwrite it; claiming a non-audit task with empty class leaves it empty.
