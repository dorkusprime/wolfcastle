# Wire ClassRepository.Validate() into daemon startup

In internal/daemon/daemon.go, after classes.Reload(cfg.TaskClasses) is called, invoke classes.Validate() and log warnings for any class keys that have config entries but no resolvable prompt file. This should be a warning, not a fatal error: the daemon starts regardless, but operators see which classes are degraded. Use the daemon's existing logger. The warning message should list the missing class keys so operators can author the missing prompts or remove the config entries.
